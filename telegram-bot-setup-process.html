Telegram Bot Project

Telegram Bot Setup Process

This page summarizes the full baseline setup completed for the Telegram bot project on the laptop. It is based on the recorded setup log and is meant to make the process easier to review visually.

Baseline status: Completed — the Telegram bot is connected to OpenClaw, routed to the aja agent, and command execution through exec is blocked for safety.

Goal

Build a Telegram bot that can be contacted directly through Telegram and connected to OpenClaw cleanly and safely.

Primary Agent

The Telegram binding was moved from main to a dedicated agent named Aja.

Safety Rule

The Telegram agent was configured to deny the exec tool, so it cannot run desktop/system commands.

Initial Setup Steps

Step 1 — Create the Telegram bot

  • Created the bot using BotFather.
  • Obtained the Telegram bot API token.

Step 2 — Back up the OpenClaw configuration

  • Backed up openclaw.json before making changes.
  • This ensures the original configuration can be restored if needed.
  • Windows path used during setup: C:\Users\jatin\.openclaw\openclaw.json

Step 3 — Check OpenClaw health before configuration

  • Ran openclaw doctor
  • Ran openclaw status
  • Confirmed there was no critical blocker before proceeding.

Phase 1 — Basic Telegram to OpenClaw Connection

The first success condition was simple: send hello to the Telegram bot and receive a reply.

Phase 1 — Step 4

  • Added Telegram as a channel in OpenClaw using openclaw channels add.
  • Selected the correct account path during setup.
  • Entered the bot token from BotFather.
  • Set DM access policy to pairing for safer initial testing.
  • Bound the Telegram account to the main agent temporarily for baseline testing.

Phase 1 — Step 5

  • Checked pending pairing using openclaw pairing list.
  • Approved the pairing using openclaw pairing approve telegram <code>.
  • Confirmed the bot could send and receive replies after approval.

Phase 2 — Create a New Agent and Move the Telegram Binding

Phase 2 — Step 1

  • Created a new OpenClaw agent named Aja.
  • Command used: openclaw agents add Aja

Phase 2 — Step 2

  • Created a separate workspace for the new agent.
  • Reused the auth profile from the main agent.
  • Skipped reconfiguring model/auth and skipped chat channel setup during the agent creation flow.

Phase 2 — Step 3

  • Checked bindings with openclaw agents list --bindings.
  • Unbound Telegram from main.
  • Bound Telegram to aja.
  • This moved the bot from the default assistant to the dedicated Telegram agent.

Phase 3 — Rules and Restrictions for the Agent

Phase 3 — Step 1

  • Set the agent identity using openclaw agents set-identity --agent aja --name "Aja" --emoji "🧠".
  • Visible identity configured as:
  • Name: Aja
  • Emoji: 🧠

Phase 3 — Step 2

  • Added an agent-level tool restriction so the Telegram bot cannot use exec.
  • File locations for openclaw.json:
  • Windows: C:\Users\<username>\.openclaw\openclaw.json
  • Linux: /home/<username>/.openclaw/openclaw.json
  • macOS: /Users/<username>/.openclaw/openclaw.json
  • Inside the target agent object in agents.list, add:
  • "tools": { "deny": ["exec"] }

This is an agent-level restriction, not a top-level tools setting.

Final Baseline Result

  • Telegram bot created and connected
  • Pairing flow tested successfully
  • Dedicated agent Aja created
  • Telegram binding moved to Aja
  • Agent identity set
  • exec denied for safety
  • First baseline of the project completed
